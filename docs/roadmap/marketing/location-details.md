# Marketing Page – Location Details

Back to [Marketing Phase](../phases/05-marketing-website.md)

## Route

`/locations/[slug]` (or `/location/[id]`)

## Objective

Present full location information and guide users toward booking intent.

## Sections

1. Media Gallery
2. Location Summary
3. Pricing & Availability Widget
4. Amenities & Rules
5. Host Snapshot
6. Similar Locations
7. Sticky Booking CTA

## Section requirements

### 1) Media Gallery
- Hero image/video
- Thumbnail strip
- Lightbox/modal behavior placeholder

### 2) Location Summary
- Title
- Area/city
- Category tags
- Short description

### 3) Pricing & Availability Widget
- Hourly / daily display
- Date + time picker UI shell
- Price estimate preview (frontend calculation)
- “Request booking” CTA (mock flow)

### 4) Amenities & Rules
- Icon list/grid for amenities
- Rules list in concise format

### 5) Host Snapshot
- Host name/avatar
- Short trust indicators placeholders

### 6) Similar Locations
- Reuse location cards component
- 3–6 related items from mock data

### 7) Sticky CTA
- Persist booking action on mobile scroll

## Data mode

Frontend mock location payload only.

## Done when

- [ ] Gallery, details, and pricing widget are complete
- [ ] Booking CTA is obvious and persistent
- [ ] Similar locations section reuses card component
- [ ] Layout is responsive and RTL-safe
