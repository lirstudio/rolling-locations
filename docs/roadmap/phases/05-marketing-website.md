# Phase 6 – Marketing Website

Back to [Roadmap Index](../../roadmap.md)

## Objective

Build the public-facing marketing and discovery pages after dashboard role flows are stable.

## Strategy

- One chat per marketing page file.
- Build each page in sections.
- Start with static/mock content and clean UX.
- Defer backend search/booking integration until a later integration milestone.

## Page files

- [Home](../marketing/home.md)
- [Locations Index](../marketing/locations-index.md)
- [Location Details](../marketing/location-details.md)
- [404](../marketing/404.md)
- [Other Required Pages](../marketing/other-required-pages.md)

## Global requirements for all marketing pages

- Hebrew + RTL first
- One primary CTA per page
- Reuse shared components/tokens from app foundation
- Mobile-first responsive behavior
- SEO basics (title/description/open graph placeholders)

## Out of scope

- Real search API
- Real booking submit
- Analytics wiring

## Demo acceptance

By end of phase, you can:
- [ ] Navigate full marketing flow (home → listings → details)
- [ ] Use polished section-based pages with consistent design system
- [ ] Hit a branded custom 404 page
